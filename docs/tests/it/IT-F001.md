# 結合試験仕様書 F001（文豪ずんだもん初期公開版）

## 1. 試験方針

本仕様書は、青空文庫コンテンツ更新、編集レビュー、VOICEVOX事前生成、静的catalog、ブラウザUI、GitHub Pages向けbuildとCIを結合した振る舞いを検証する。純粋関数内の細かな境界値はUTへ委ね、ITではartifact間の参照、ブラウザAPI境界、失敗時のロールバック、表示・通信・証跡の同期を確認する。

- 外部取得とVOICEVOX ENGINE接続は明示的な更新試験だけで使用し、通常build・閲覧試験は固定fixtureと生成済み資産だけを使用する。
- 時計、取得応答、VOICEVOX応答、Audio要素、MediaQueryList、IntersectionObserverは試験用アダプタで制御する。
- 一時directoryから正常時だけ成果物を昇格し、失敗時は直前の正常artifactのhashが変わらないことを共通不変条件とする。
- ブラウザ試験ではconsole error、CSP violation、request URL、Cookie、Storage、音声状態、アクセシビリティtreeを証跡へ保存する。
- 入力fixtureの版、source hash、変換器版、VOICEVOX設定、Vite base、ブラウザ名を失敗時に再現情報として出力する。
- 各ITは固有の一時workspace/cacheとin-memory network/audio adapter、固定UTC時計を使用し、adapter・module・browser contextを終了時にresetする。固定portを共有せず、単独実行・逆順実行でも同じ結果になることを確認する。
