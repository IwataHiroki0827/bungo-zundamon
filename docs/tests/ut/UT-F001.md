# 単体試験仕様書 F001（文豪ずんだもん初期公開版）

## 試験方針

- `DD-F001.md`の`FUN-F001-001`〜`FUN-F001-042`を各1ケース以上で網羅する。
- 各ケースは正常入力に加え、仕様上の境界値または異常入力を同一ケース内のパラメータ試験として実行する。
- 例外・検証失敗時は、部分的な公開、既存artifactの破壊、秘密情報の表示、外部originへの通信がないことも確認する。
- 自動化対象はVitest（必要に応じてjsdom、fixture、fake timer、fetch/audio mock、一時directory）で実装し、結果は`PASS`、`FAIL`、`BLOCKED`のいずれかで記録する。
- 各行の正常・境界・異常variantへ`UT-F001-NNN.normal`、`.boundary.<name>`、`.error.<name>`のsubcase IDを付け、結果・入力fixture・理由をsubcase単位で保存する。1 variant失敗を行全体の単一結果へ埋没させない。
- 各subcaseは固有の一時workspace/cacheとin-memory adapterを使い、fake clock・module・DOM・Audio・fetch spyをafterEachでresetする。固定portを共有せず、単独実行と順序変更実行で同じ結果になることを確認する。
