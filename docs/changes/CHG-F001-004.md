# CHG-F001-004: 演出低減設定の効果説明

| 項目 | 内容 |
|---|---|
| 発生日 | 2026-07-19 |
| 起票契機 | ユーザー試験で「演出を控えめにする」の効果が分からないと指摘 |
| レベル | design（REQ-F001-022の受け入れ表示を具体化） |
| 影響 | REQ-F001-022、DES-F001-011、FUN-F001-025、UT-F001-025、IT-F001-011、QT-F001-015、T-013 |

## 原因

- 実際に止まるのはページ切替時の約420msの移動と、再生アイコンの反復アニメーションだけであり、差が短く小さい。
- 設定欄は次の操作名しか表示せず、現在状態と停止対象を説明していなかった。
- OSの`prefers-reduced-motion`が有効な場合はサイト操作で標準へ戻せないが、従来ボタンは操作可能に見え、端末設定優先の理由も表示しなかった。

## 修正

- 標準時は「演出：標準」「ページ切替と再生アイコンが動きます」と表示する。
- サイト設定低減時は「演出：控えめ」「ページ切替と再生アイコンの動きを停止中」と表示する。
- OS reduced時は「端末設定により動きを停止中」と表示し、上書き不能なボタンを無効化する。
- `aria-label`は現在状態ではなく次の操作を伝え、端末設定優先時は理由を伝える。
