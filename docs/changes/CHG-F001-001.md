# CHG-F001-001: テスト可能な設計契約への補強

| 項目 | 内容 |
|---|---|
| 発生日 | 2026-07-18 |
| 起票契機 | テスト仕様レビューパネルでHigh 3件・Medium 12件を検出 |
| レベル | design（承認済み要求の達成方法を変更せず、実装・試験可能な契約へ具体化） |
| 影響範囲 | FD-F001、DD-F001、UT-F001、IT-F001、QT-F001、実機browser証跡template、T-003 |
| 承認 | requirement変更なし。designレベルのため追加の要求承認ゲートは発生しない |

## 変更理由

- 音声生成1件失敗時に全成果物を巻き戻す設計が、SRS-F001の「失敗台詞だけ公開対象外」と矛盾していた。
- parse済みJSONだけを受ける関数では、raw catalog byte上限を検査できなかった。
- productionのSSRF防止でloopbackを拒否する一方、IT正常系がlocalhost fixture serverを要求していた。
- テストケースに判定定数、操作語義、実browser・Actions・画像入力由来の証跡が不足していた。
- 承認前QTが本番repositoryのpublic化を要求し、リリース承認ゲート④と循環していた。
- VOICEVOX ENGINE版のruntime照合と、公式書誌取得のproduction transport契約が欠落していた。

## 機械抽出した影響

`docs/traceability/tm.json`から、上記DESを参照するFUN/UT/ITと対応REQのQTを抽出し、`docs/changes/changes.yaml`の`affected`へ記録した。既存T-003を影響文書の修正・再レビュータスクとして継続使用する。実装前のため再実施対象コードはなく、テスト仕様レビューと`trace_check`を再検証とする。

- [x] `trace_check`が対応漏れ0件となる。
- [x] テスト仕様承認ゲート③をQ-004として登録する。
