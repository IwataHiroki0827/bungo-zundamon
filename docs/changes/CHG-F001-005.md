# CHG-F001-005: 手動ブラウザ試験範囲とhosted実施時期の見直し

## 現状の問題

- 変更前のREQ-F001-030はPC Chrome/Edge/Firefox/Safari、Android Chrome、iOS Safariの6環境をすべて手動必須としていた。
- Chromium/Firefox/WebKit、Chrome stable、Edge stableでは自動E2E 65件を既に実施していたが、変更前要求では実機の代替にならなかった。
- hosted Actionsはワークフローを含むリリース候補をcommit/pushした後でなければ実施できず、未pushの作業ツリーに対して先にユーザー試験を依頼した手順順序が誤っていた。

## 推奨案

- 手動必須をWindows Chrome/EdgeとiOS Safariへ絞る。
- Firefox/WebKitとAndroid相当viewportはPlaywrightの継続試験で担保し、リスクベースで実機試験を追加できる扱いにする。
- hosted Actionsはリリース候補commitをGitHubの非公開feature branchへpushした後、実行可能なrun URLをProjectFactoryへ表示して確認する。

## 承認結果と反映内容

2026-07-19にQ-007で推奨案が承認された。次を要求・設計・試験仕様へ反映した。

- 手動必須はWindows Chrome、Windows Edge、iOS Safariの3環境とする。
- Firefox、desktop Safari相当のWebKit、Android相当viewportはPlaywrightの自動継続試験で担保する。
- 自動失敗、ブラウザ固有リスク、表示・音声差異がある場合は該当実機試験を追加する。
- hosted Actionsは候補commitをprivate feature branchへpushした後に実施し、run URL、artifact digest、対象commitを証跡化する。

## 再レビュー

初回pf-reviewerはHigh 2件、Medium 3件、Low 1件で差し戻した。指摘を受け、次を補強した。

- ゲート④前は`PAGES_DEPLOY_ENABLED`未設定またはfalseを必須とし、main成功だけではdeployできないfail-closed event matrixを追加した。
- `HostedBuildEvidence`を同一repository、run、event、ref、head SHA、workflow、artifact、catalog hash、deployment不在、Pages hash不変へ結合した。
- `BrowserRiskDecision`のtriggerと追加実機の決定・解消規則を定義した。
- negative hosted runを候補commitとは別の検証専用commitへ分離した。
- 手動・自動・hosted証跡テンプレートを新契約へ更新した。
- deploy変数を承認対象SHAへ一回限りで拘束し、事後証跡で変数無効化まで検証する契約を追加した。
- ブラウザ証跡の鮮度は経過時間ではなく候補commit/catalog hash一致で判定するよう明確化した。
