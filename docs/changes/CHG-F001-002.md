# CHG-F001-002: 公式書誌ZIPの安全な取得・展開

| 項目 | 内容 |
|---|---|
| 発生日 | 2026-07-18 |
| 起票契機 | 実公式書誌取得前のURL・media・byte数照合 |
| レベル | design（対象作品・公開内容を変えず、実在する公式配布形式へ適合） |
| 公式配布 | `list_person_all_extended_utf8.zip`、`application/zip`、2026-07-18確認時2,092,030 bytes |
| 固定entry | `list_person_all_extended_utf8.csv`、同確認時17,153,006 bytes |
| 影響 | DES-F001-003/004/017/019、FUN/UT-F001-008/041、IT-F001-001、QT-F001-003/007、T-004 |

## 原因

承認済み設計は「公式書誌CSVを8MiB以下で取得する」としていたが、青空文庫の公開URLはZIP配布だけである。拡張子なしURLも同じZIPを返し、`.csv` URLは404だった。現実装はCSV media typeだけを許可するため、正常な公式配布物を必ず拒否する。

## 変更内容

- URLを公式ZIPの完全一致へ固定する。
- compressed bytesは従来どおり8MiB以下とする。
- 固定CSV entryの展開上限を32MiB、展開比を20以下とする。
- 単一entry、暗号化なし、multi-diskなし、store/deflateだけ、central/local header整合、CRC-32を検証する。
- filesystemへ任意entryを展開せず、検証済みCSVだけをmemory上でparseする。
- ZIPとCSVの各hash/byte数をsnapshotとprovenanceへ残し、同じstagingからatomic promotionする。
- `ProvenanceManifest.bibliography`と全作品の由来情報へ同一のZIP/CSV hash・byte数・固定entry・schema版を必須で引き継ぐ。
- 初期版の杜子春ID誤記も、公式新字新仮名版`043015`へ補正する。

## 機械抽出した影響

`docs/traceability/tm.json`でDES-F001-003/004/017/019を参照するFUN/UT/ITを抽出し、直接変更するFUN/UT-F001-008/041とIT-F001-001を修正対象とした。対応要求の資格試験からQT-F001-003/007を更新する。実装タスクT-004を`doing`へ戻し、再受け入れ対象とする。

## 実装時疎通証跡

2026-07-18にNode標準HTTPSで確認したところ、User-Agent未指定要求はHTTP 302と`http://mirror.aozora.gr.jp/...`を返したため追従せず拒否した。環境値・入力値を含まない固定User-Agent `bungo-zundamon/0.1 (ProjectFactory content fetcher)`を付けた同一URLへの初回要求は、redirectなしでHTTP 200・`application/zip`を返した。Production transportによる取得・安全検証後の値は、ZIP 2,092,030 bytes / SHA-256 `069280a4aa17ac2d382605dc4d189d1bde8762973324cd832e4ddf391dc82af0`、CSV 17,153,006 bytes / SHA-256 `28d37d3d5f94e4d3c8f7838944b076ff854c47f8cb29c533fd69ada7cd2920a0`、19,492行である。そこから`000127`（羅生門）、`000092`（蜘蛛の糸）、`043015`（杜子春）を選定できた。
