# 要求仕様書 F002（継続コンテンツ拡充）

## 概要

公開済みF001の芥川龍之介3作品と59台詞・音声を維持したまま、複数作者をデータ駆動で追加できる構造へ拡張する。初回追加バッチとして宮沢賢治「よだかの星」「どんぐりと山猫」「注文の多い料理店」の`「」`候補を全件編集レビューし、承認済み台詞のVOICEVOXずんだもん音声を差分生成してGitHub Pagesへ静的配信できる状態にする。以後は1作者・代表3作品を1フィーチャーとして同じ権利・品質・容量・セキュリティゲートで継続追加する。

## 対象範囲

### 対象

- 単一作者・3作品のハードコードを廃止した複数作者catalog、route、UI
- 宮沢賢治と代表3作品の書誌、台詞、VOICEVOXずんだもん音声
- 全候補編集レビュー、読み補正、音声差分生成、容量preflight
- 作者・作品・音声ごとの出典・クレジット・権利証跡
- F001品質を維持するセキュリティ、アクセシビリティ、ブラウザ回帰
- 1作者・3作品単位の継続投入手順と受入証跡

### 対象外

- `『』`等、現行`「」`以外を対象にする抽出仕様拡張
- 宮沢賢治の上記3作品以外、次回候補「雪渡り」、次作者候補の実収録
- 収益化、広告、スポンサー、課金、会員登録、利用者投稿
- 実行時の青空文庫取得、VOICEVOX ENGINE・外部TTS API接続
- VOICEVOXソフトウェア、音声モデル、第三者二次創作素材そのものの再配布
- GitHub Pages以外の音声CDNまたは配信基盤への移行

## 品質・運用条件

- コンテンツ取得とVOICEVOX音声生成は明示更新時だけ行い、通常buildとブラウザ実行時のネットワーク取得を禁止する。
- 作品は小さい順に「よだかの星」→「どんぐりと山猫」→「注文の多い料理店」と進め、作品ごとにレビュー・容量・音声・受入を完結させる。
- 文学作品中のいじめ、死、暴力、宗教、侮辱表現だけを理由に実発話を機械除外しない。ただし作品中の表現・作者・作品・出典を明示し、音源・キャラクター規約の実際の禁止用途は除外する。
- 原典SHA-256または公式規約が変わった場合は、影響する権利判断、レビュー、読み補正、音声、クレジットを再確認する。
- 100 MiBは104,857,600 bytes、500 MiBは524,288,000 bytes、750 MiBは786,432,000 bytesとして境界試験する。
- GitHub Pagesの公開物・source repositoryに用いる1 GBは、安全側で10進の1,000,000,000 bytesとする。公開物は最終Pages artifactの全ファイルを再帰的に数え、source repositoryは`.git/objects`の実測値と当該バッチで新たにGit object化するファイルの見込値を重複なく数える。
- 公開物の750 MiB超停止基準をPages上限への安全余裕として優先し、source repositoryも750,000,000 bytes以上で警告、1,000,000,000 bytes以上で停止する。
- F001の既承認content/audioを変更する必要が生じた場合は、F002へ便乗せず変更管理を通す。
