## 1. アプリシェル・公開データ

### FUN-F001-001: parseRoute(hash: string) -> Route [DES-F001-001]

- **入力**: `location.hash`。長さ256以下の文字列。
- **出力**: `home | author | credits | notFound`の判別共用体。作者slugは固定allowlistだけを受理する。
- **エラー処理**: decode失敗、制御文字、未知slugは`notFound`。外部URLへ遷移しない。

### FUN-F001-002: renderRoute(root: HTMLElement, route: Route, catalog: Catalog) -> void [DES-F001-001][DES-F001-010]

- **入力**: 固定root、検証済みroute/catalog。
- **出力**: 対応pageを`replaceChildren`で描画し、main見出しと現在位置を更新する。
- **エラー処理**: 不変条件違反は安全なページ内エラーへ変換し、既存AudioControllerを停止する。

### FUN-F001-003: loadCatalog(baseUrl: URL, signal?: AbortSignal) -> Promise<Catalog> [DES-F001-002][DES-F001-013][DES-F001-015][DES-F001-019]

- **入力**: 同一オリジンかつ`/bungo-zundamon/`配下と検証済みbase。
- **出力**: `content/catalog.json`をstream読込し、raw UTF-8 byte数を測定してFUN-F001-004で検証したcatalog。
- **エラー処理**: `MAX_CATALOG_BYTES`を1byteでも超えた時点で読込を中止する。UTF-8 fatal decode後だけJSON parseし、network、HTTP、decode、JSON、schemaエラーを`CatalogLoadError`へ正規化して再読込導線を表示する。

### FUN-F001-004: validateCatalog(value: unknown, sourceByteLength: number) -> ValidationResult<Catalog> [DES-F001-002][DES-F001-013]

- **入力**: JSON parse結果とparse前に測定済みのraw UTF-8 byte数。
- **出力**: schemaVersion、ID一意性、3作品、参照整合、相対path、候補集計を検証した型付きcatalog。
- **エラー処理**: 製品上の台詞件数上限は設けず、8MiB raw byte、32,768 scalar/文字列、depth 64、3作品固定、容量予算で資源消費を制限する。`futureExpansionPolicy`、公開・編集除外・音声失敗除外の候補集計、共有音声参照を含む全違反を安全な理由コードで返し、部分的catalogを公開しない。

## 2. 原典選定・抽出・レビュー

### FUN-F001-005: selectEligibleWorks(rows: BibliographyRow[]) -> WorkCandidate[] [DES-F001-003]

- **入力**: schema検証済み公式書誌行。
- **出力**: 人物`000879`、著者、著作権なし、公開中、日本語原著を満たす安定ソート済み候補。
- **エラー処理**: 必須列欠落・未知値は対象外にして診断。0件なら工程全体を失敗させる。

### FUN-F001-006: resolveEdition(candidates: WorkCandidate[], rules: EditionRule[]) -> SelectedWork[] [DES-F001-003]

- **入力**: 候補と作品IDを明記した版対応規則。
- **出力**: 羅生門、蜘蛛の糸、杜子春の各1版と選択理由。
- **エラー処理**: タイトル一致だけの自動統合、循環規則、同一ID重複、allowlist外作品を拒否する。

### FUN-F001-007: fetchAozoraSources(works: SelectedWork[], outputDir: Path) -> Promise<SourceRecord[]> [DES-F001-004][DES-F001-017][DES-F001-019]

- **入力**: `https://www.aozora.gr.jp/`の固定path prefixを満たすURL、実体pathがworkspace内でreparse pointを含まない一時directory、CLIが固定生成する`ProductionAozoraTransport`。transportはresolverとpinned socket factoryだけを依存として持つ。
- **出力**: raw bytes、取得日時、raw SHA-256、HTTP charset（応答で省略時は`null`）、選定済み書誌charsetを持つsource record。meta charsetの走査とdecodeは行わない。
- **エラー処理**: production transport自身がredirectと環境proxyを無効化する。接続前に名前解決を1回だけ行いprivate/link-local/loopback/予約addressを拒否し、その検証済みaddressへ作品ごとに直列で1回だけ接続し、自動retryしない。公式hostnameに対するTLS証明書検証を維持し、不正証明書とhostname mismatchを拒否する。origin/path prefix、8MiB/作品、15,000ms、content-type、HTTP状態を検証する。testはtransport本体を差し替えず、resolver/pinned socket/TLS factoryだけをfake化してURL・pin先・Host/SNI・証明書検証・要求回数・同時実行数・proxy/redirect無効化を観測する。失敗時は一時directoryを採用しない。

### FUN-F001-008: buildProvenance(source: SourceRecord, metadata: AozoraMetadata) -> Provenance [DES-F001-004][DES-F001-012][DES-F001-017]

- **入力**: hash確認済み原典と書誌。
- **出力**: 図書カード、底本、入力者、校正者、取得日、加工、CC BY 4.0変更表示を含む由来情報。
- **エラー処理**: 安定リンク、取得元・取得日時・source hash・tool版・空でない加工内容の欠落は公開不可として返す。

### FUN-F001-009: extractDialogueCandidates(source: DecodedSource, workId: string) -> ExtractionResult [DES-F001-005][DES-F001-019]

- **入力**: FUN-F001-040でcharset検証・decode済みの青空文庫XHTMLとallowlist済み作品ID。
- **出力**: 本文順の`RawCandidate`、周辺文、ruby token、source anchor、診断。表示文・読み上げ文・candidate IDはまだ生成しない。
- **エラー処理**: parser error、本文欠落、括弧不整合は理由コード付きnonzero相当結果。HTML文字列を実行しない。

### FUN-F001-010: tokenizeAozoraBody(document: Document) -> TextToken[] [DES-F001-005]

- **入力**: inert DOM parserで生成したDocument。
- **出力**: 本文コンテナ内のtext/ruby/lineBreak token。
- **エラー処理**: script、style、navigation、注記、未知要素は実行せず除外し、未知の本文要素は診断へ記録する。

### FUN-F001-011: normalizeDisplayText(tokens: TextToken[]) -> string [DES-F001-006]

- **入力**: 1候補分token。
- **出力**: 原文表示を保ったNFC文字列。
- **エラー処理**: NUL・孤立surrogate・長さ上限超過を拒否する。

### FUN-F001-012: normalizeSpeechText(tokens: TextToken[], rules: SpeechRules) -> string [DES-F001-006]

- **入力**: tokenと版付きruby・外字・空白規則。
- **出力**: VOICEVOX入力用の決定的文字列。
- **エラー処理**: 読み不明外字、空文字、設定版不明は`NormalizationError`。

### FUN-F001-013: createCandidateId(workId: string, rawSourceSha256: Sha256, rawTokenRange: TokenRange, extractorVersion: string, normalizerVersion: string, normalizedTextHash: Sha256) -> CandidateId [DES-F001-002][DES-F001-006]

- **入力**: allowlist済み作品ID、検証済みraw source hash、byte/token境界が一意なrange、semver抽出器・正規化器版、表示文と読み上げ文を順序付きでhashした値。
- **出力**: 入力tupleのcanonical encodingをSHA-256化したURL安全な安定ID。正規化工程で`Candidate.candidateId`へ設定し、承認後の公開catalogでは同じ値を`dialogueId`として使う。
- **エラー処理**: 入力不正を例外で停止し、ランダムfallbackを作らない。raw source、range、抽出器、正規化器、正規化文のいずれかが変わればIDも変わり、FUN-F001-014は旧IDのレビューを孤立レビューとして失敗させる。

### FUN-F001-014: applyEditorialReview(candidates: Candidate[], reviews: ReviewRecord[]) -> ReviewResult [DES-F001-007]

- **入力**: 抽出候補と候補IDに厳密一致する人手レビュー。
- **出力**: approved、rejected、pending、理由別集計。
- **エラー処理**: 未知ID、重複レビュー、必須理由なし、pending残存を失敗とし、暗黙承認しない。

### FUN-F001-015: buildPublicCatalog(input: ReviewedContent, voice: VoiceGenerationResult, assets: AssetManifest) -> Catalog [DES-F001-002][DES-F001-007][DES-F001-013]

- **入力**: pendingなしのレビュー結果、台詞単位失敗を含む音声生成結果、hash検証済み資産。
- **出力**: 編集approvedかつ音声成功した台詞だけ、共有`audioId`参照、公開・編集除外・音声失敗除外の候補集計、初期3作品後の拡張条件・権利再確認・段階追加方針を持つ`futureExpansionPolicy`を含む公開catalog。canonical UTF-8 serialize後のbyte数をFUN-F001-004と同じ上限で再検証する。
- **エラー処理**: pending混入、理由なし音声除外、件数不一致、孤立資産、絶対URL、重複IDでbuildを停止する。失敗台詞のreview/candidate artifactは削除しない。

## 3. 音声生成・再生

### FUN-F001-016: createVoiceCacheKey(text: string, config: VoiceConfig) -> VoiceCacheKey [DES-F001-006][DES-F001-008]

- **入力**: 正規化読み上げ文、ENGINE版、固定speaker UUID・speaker名・style ID・style名、音声設定。
- **出力**: 全設定を含むSHA-256 cache key。
- **エラー処理**: 未固定版、範囲外設定、空文字を拒否する。

### FUN-F001-017: generateVoiceAssets(items: SpeechItem[], client: VoicevoxClient, cacheDir: Path) -> Promise<VoiceGenerationResult> [DES-F001-008][DES-F001-017][DES-F001-019]

- **入力**: approved項目、`http://127.0.0.1:50021`または`http://[::1]:50021`だけを受理する`ProductionVoicevoxClient`、repositoryへ固定した`VoiceConfig`、実体path確認済みworkspace内cache。clientは本番実装のまま、testでは低水準connectorだけをfake化する。
- **出力**: cache再利用を含む検証済みWAVと、candidate/audio ID別の成功・失敗理由を持つ`VoiceGenerationResult`。
- **エラー処理**: 最初に`/version`を1回取得して妥当な文字列であることと期待`engineVersion`への完全一致を検証し、続いて`/speakers`を1回取得して期待speaker UUID・speaker名`ずんだもん`・style ID・style名の完全一致を検証する。cache missごとにaudio queryとsynthesisを直列で各1回だけ要求し、自動retryしない。台詞単位のtimeout・非WAV・0byteはそのaudio IDだけを一時領域から破棄して理由付き失敗へ記録し、他の検証済みcacheはkey単位でatomic promotionする。`/version`欠落・malformed・不一致、話者4項目不一致、hostname、別port、redirect、環境proxy、loopback以外の接続先、symlink/junction/reparse point、client設定・workspace境界・manifest不変条件の異常はstage全体を拒否する。公開directoryへ直接書かない。

### FUN-F001-018: verifyAssetBudget(manifest: AssetManifest) -> BudgetReport [DES-F001-008][DES-F001-014]

- **入力**: build出力全資産のpath・size・hash。
- **出力**: 合計、媒体別、重複、最大ファイルの報告。
- **エラー処理**: 合計750,000,000 byte以上、104,857,600 byte以上の単一ファイル、参照欠損、異なるasset path間のhash重複はhard fail。同一`audioId`を複数台詞が参照する共有は許可する。合計500,000,000 byte以上は警告を返す。

### FUN-F001-019: AudioController.play(item: Dialogue, trigger: HTMLButtonElement) -> Promise<PlayerState> [DES-F001-009][DES-F001-014][DES-F001-015]

- **入力**: 検証済み台詞とtrustedなbutton操作。
- **出力**: 前音声を停止し対象1件だけを遅延取得・再生した状態。
- **エラー処理**: 自動再生を行わず、URL解決・HTTP・decode・play rejectionをFUN-F001-021へ渡す。

### FUN-F001-020: AudioController.control(action: 'toggle' | 'stop', dialogueId?: DialogueId) -> PlayerState [DES-F001-009]

- **入力**: `toggle`は操作対象ID、`stop`は省略時に現在音声を対象とする。
- **出力**: 再生中の`toggle`は位置を保持した`paused`、pausedの`toggle`は同じ位置から`playing`、`stop`だけが`currentTime=0`の`stopped`とし、button状態とlive regionを同期する。
- **エラー処理**: 未知IDは状態不変。例外をUIイベントループへ投げない。

### FUN-F001-021: presentAudioError(itemId: DialogueId, cause: unknown) -> void [DES-F001-009][DES-F001-019]

- **入力**: 対象IDと内部原因。
- **出力**: 固定日本語理由、再試行button、`aria-live`通知。
- **エラー処理**: 生URL、stack、環境値をDOMへ出さず、他カードとnavigationを維持する。
