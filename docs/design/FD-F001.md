# 機能設計書 F001（文豪ずんだもん初期公開版）

## 1. 設計方針

本アプリはViteとVanilla TypeScriptで構築する完全静的Webアプリとする。通常閲覧時はGitHub Pagesの同一オリジンに配置したHTML、CSS、JavaScript、JSON、画像、音声だけを読み、青空文庫取得とVOICEVOX生成は公開物の外側にある明示的な更新工程へ分離する。画面はhash routeを使い、`/bungo-zundamon/`直下の再読込で404を発生させない。

信頼境界は「取得した原典」「編集者のレビュー記録」「公開catalog」の3段階に分ける。公開buildはschema検証済みかつ`approved`の台詞だけを入力にし、文字列は`textContent`または属性専用APIで描画する。ランタイムに認証、Cookie、Web Storage、解析、フォーム、外部APIを持たない。

## 2. 画面・コンテンツ設計

### DES-F001-001: アプリシェルとhash route [REQ-F001-001][REQ-F001-002][REQ-F001-003][REQ-F001-020][REQ-F001-025]

`#/`をトップ、`#/authors/akutagawa-zunnosuke`を作者ページ、`#/credits`をクレジットページとする。未知routeはトップへの安全な導線を持つ404表示にする。全routeは単一`index.html`から描画し、ブラウザ再読込時のサーバー側rewriteを不要にする。サイト名、機能説明、作者一覧、非公式表記をトップで確認できる。

### DES-F001-002: 公開catalogとmanifestの契約 [REQ-F001-004][REQ-F001-005][REQ-F001-011]

公開データ`public/content/catalog.json`はraw UTF-8で8MiB以下とし、`schemaVersion`、作者、3作品、台詞、共有音声asset、出典、クレジット参照、`futureExpansionPolicy`を持つ。作品は`workId/title/cardLink/source`、台詞は`dialogueId/order/displayText/speechText/audioId/sourceAnchor/review`、音声assetは`audioId/path/hash/bytes/duration`を必須とする。同じ読み上げ文・設定の台詞は同一`audioId`を参照できる。`PublicAssetPath`は同一オリジン相対path、`TrustedExternalLink`はbuild時allowlistを通ったHTTPSリンクとして型を分離する。候補数が公開採用数、編集理由付き除外数、音声生成理由付き除外数の合計に一致しない場合や、初期3作品後の拡張条件・権利再確認・段階追加方針が欠落した場合はbuildを失敗させる。

### DES-F001-003: 対象作品選定と版対応 [REQ-F001-005][REQ-F001-006][REQ-F001-007]

公式書誌fixtureから人物ID`000879`、役割`著者`、作品著作権`なし`、公開中、日本語原著を満たす作品だけを候補化する。翻訳者登録と`Afterlife`を除外する。タイトル一致だけでは重複扱いせず、作品ID対応表で同一と確認した版だけを統合し、新字新仮名を優先、なければ明示したfallbackを用いる。初期版のallowlistは「羅生門」「蜘蛛の糸」「杜子春」の採用作品IDに固定する。

### DES-F001-004: 原典取得と由来情報 [REQ-F001-016][REQ-F001-024][REQ-F001-029]

ネットワークを使う`content:update:bibliography`/`content:update:sources`と、固定fixtureだけを読む通常buildを分離する。公式書誌と作品XHTMLの取得は同じ`ProductionAozoraTransport`契約を使い、公式HTTPS originと用途別path prefix、TLS証明書・hostname、DNS pin、redirect/proxy無効化、media type、byte上限、1対象1要求・自動retryなしを検証する。取得工程はraw bytesのままSHA-256を算出し、図書カードURL、XHTML URL、取得日時、宣言charset、底本、入力者、校正者、書誌データ版、加工内容をprovenanceへ保存する。青空文庫のShift_JIS/UTF-8だけを明示decodeする。HTTP `Content-Type`、raw bytesから読んだmeta、書誌のcharsetはそれぞれ省略可能だが、存在する宣言はすべて一致させ、全宣言欠落を拒否する。不一致とdecode不能は公開不可にする。通常buildや閲覧時には取得先へ通信しない。

### DES-F001-005: 台詞候補抽出 [REQ-F001-008]

FUN-F001-040でraw bytesを宣言charsetどおりdecodeしたXHTMLだけをDOM化し、本文コンテナだけを走査する。rubyは表示本文と読みを保持した中間tokenへ変換する。`「」`の深さを状態として管理し、改行を跨ぐ台詞、同一段落の複数台詞、内側の`『』`を欠落させない。注記、書誌、ナビゲーション、脚注は走査対象外とし、括弧不整合は候補を推測せず診断へ記録する。

### DES-F001-006: 表示文・読み上げ文・安定ID [REQ-F001-010][REQ-F001-011][REQ-F001-012][REQ-F001-029]

表示文は原文の意味と表記を保ち、読み上げ文はruby読み、外字置換表、空白・改行規則を版付きの純粋変換として適用する。候補IDは作品ID、raw source hash、token range、抽出器版、正規化器版、正規化文hashから決定的に生成する。いずれかが変わった候補へ旧レビューを自動転用せず、孤立レビューとして再確認を要求する。音声cache keyは読み上げ文、VOICEVOX ENGINE版、スタイルID、速度・抑揚等の設定を含み、同一条件を再生成しない。

### DES-F001-007: 編集レビューと公開allowlist [REQ-F001-005][REQ-F001-009]

候補ごとに`approved/rejected/pending`、周辺文、判断理由、確認者、確認日、規約確認を記録する。発話ではない内心・引用・語句例、孤立表示で政治宗教・攻撃・残虐表現等と誤解される候補、規約禁止用途に触れる候補は公開しない。公開catalog生成は`approved`だけを固定allowlistとして採用し、`pending`が1件でもあれば失敗する。

### DES-F001-008: VOICEVOX事前生成 [REQ-F001-012][REQ-F001-023][REQ-F001-029]

音声生成は開発者が明示実行する更新コマンドだけが`127.0.0.1:50021`または`[::1]:50021`のVOICEVOX ENGINEへ接続する。hostname、任意port、redirect、環境proxyを許可しない。ENGINEの`/version`応答をrepositoryへ固定した期待ENGINE版と完全一致させ、続いて`/speakers`応答を期待speaker UUID・speaker名`ずんだもん`・style ID・style名と完全一致させてから生成する。ENGINE版、4項目の話者同一性、音声設定値をmanifestに固定し、生成音声のhash、長さ、byte数を記録する。音声生成は直列・自動retryなしとし、cache missごとにaudio queryとsynthesisを各1回だけ要求する。生成前に候補件数・文字数・推定秒数・推定bytesをpreflightし、容量停止基準を超える場合は生成しない。台詞単位のtimeout・破損WAV等はcandidate IDと理由を失敗manifestへ記録し、その台詞だけを公開catalogから除外する。成功した検証済みcacheと編集レビュー済み文字artifactは維持し、他の台詞まで巻き戻さない。client設定、ENGINE版、話者同一性、workspace境界、manifest整合性の異常はstage全体を停止する。通常buildはENGINEへ接続せず、成功assetと理由付き除外の整合を検査して公開tree全体をatomic promotionする。

## 3. UI・音声設計

### DES-F001-009: 単一音声プレイヤー [REQ-F001-013][REQ-F001-014][REQ-F001-015][REQ-F001-021]

利用者のbutton操作後に対象音声だけを遅延読込する。Audio要素は1インスタンスとし、別台詞の再生前に現在音声を停止して位置を戻す。同一buttonは再生中なら位置を保持して一時停止し、再操作でその位置から再開する。明示的な停止操作だけが位置を0へ戻す。`aria-pressed`、日本語状態文、アイコンとテキストで状態を通知する。404、decode、再生拒否はカード単位の再試行可能エラーにし、画面全体を停止しない。

### DES-F001-010: 一覧UI・レスポンシブ・アクセシビリティ [REQ-F001-001][REQ-F001-002][REQ-F001-003][REQ-F001-004][REQ-F001-020][REQ-F001-021]

トップ→作者→再生を3操作以内にする。見出し階層、リスト、link、buttonをセマンティックに用い、44 CSS px以上の操作領域、可視focus、十分なcontrast、状態用`aria-live`を提供する。390×844、844×390、1440×900で横溢れと重なりを防ぎ、作品ごとの台詞は段階的に表示して長大一覧を扱う。

### DES-F001-011: 文豪サムネイルと演出低減 [REQ-F001-018][REQ-F001-022]

坂本アヒル氏配布素材をSSS条件内で改変した文豪風ずんだもんを静的画像として配置し、特定の芥川写真は入力・合成しない。入力素材allowlist、各入力hash、編集元、変換手順、出力hash、確認者、確認日を`ArtworkProvenanceManifest`へ固定し、allowlist外画像や由来不明入力をbuildで拒否する。元PSDを公開せず、素材版とhashを管理記録へ残す。再生時は墨粒・紙片・音波の短い演出を行うが、`prefers-reduced-motion`またはメモリ内の演出低減設定では移動・反復を停止し、状態情報は静的に残す。設定は追跡防止のため永続化しない。

### DES-F001-012: 出典・クレジット・利用条件 [REQ-F001-016][REQ-F001-017][REQ-F001-019][REQ-F001-028][REQ-F001-030]

全ページのfooterへ`VOICEVOX:ずんだもん`と「東北ずん子・ずんだもんプロジェクトの非公式ファンサイトです」を置く。1操作以内のクレジットページに青空文庫作品出典、書誌データだけに対するCC BY 4.0の表示・リンク・変更表示（本文を同ライセンスと誤表示しない）、キャラクター、`立ち絵：坂本アヒル`、素材README版・hash・元PSD非同梱、依存物、規約確認日を掲載する。無料・広告なし・課金なし・フォーム/追跡なし・問い合わせ先方針に加え、「収録作品は日本法を基準に権利状態を確認しており、日本国外での権利状態を一律に保証しない」旨を明示する。これらを`LicenseManifest`の必須schemaとし、欠落または規約確認期限切れならbuild/releaseを停止する。将来収益化は別の権利再審査なしに有効化しない。

## 4. セキュリティ・性能・公開設計

### DES-F001-013: ランタイムセキュリティとプライバシー [REQ-F001-024][REQ-F001-027][REQ-F001-028]

CSPは`default-src 'self'`を基礎にscript、style、img、media、connectを`'self'`へ限定し、`object-src/base-uri/form-action/frame-src`を`'none'`とする。GitHub Pagesのmeta CSPでは`frame-ancestors`が無効で、親frameへの埋込みを応答headerで禁止できない点は残余リスクとして記録する。inline script、イベント属性、`eval`、`new Function`、未検証`innerHTML`を使わない。`PublicAssetPath`は相対pathだけを許可し、protocol相対、絶対URL、`..`、制御文字を拒否する。`TrustedExternalLink`はHTTPS、資格情報なし、既定port、固定origin/path prefixだけを許可し、新規tabでは`rel="noopener noreferrer"`を固定する。Cookie、LocalStorage、sessionStorage、Service Worker、外部送信を実装しない。

### DES-F001-014: 遅延読込と容量予算 [REQ-F001-013][REQ-F001-023]

初期表示ではcatalogと表示に必要なサムネイルだけを取得し、音声は操作時に1件ずつ取得する。build前preflightとbuild後の両方で件数・推定/実測秒数・全ファイルと音声別サイズを集計する。合計はdecimal byteで500,000,000以上を警告、750,000,000以上をhard fail、単一ファイルはbinaryの100MiB（104,857,600 byte）以上をhard failとする。同一`audioId`を複数台詞が共有することは許可し、異なるpathに同一音声hashを複製することだけを拒否する。

### DES-F001-015: GitHub Pagesパス契約 [REQ-F001-025]

生成側はVite `base='/bungo-zundamon/'`を唯一のbaseとし、HTMLのbundle URLはViteへ委ねる。catalog内の`audioPath/imagePath`は先頭slashなしの公開root相対値（例:`audio/F001/...wav`）で保存する。消費側は`new URL(path, import.meta.env.BASE_URLを絶対化したbase)`相当の共通関数だけでURL化し、文字列連結を禁止する。CIは`dist/index.html`、bundle、catalog、全参照資産を`/bungo-zundamon/`基準で実際に解決して404相当を検出する。

### DES-F001-016: CIと最小権限デプロイ [REQ-F001-026]

pull requestとpushのbuild jobは`permissions: contents: read`で型検査、lint、UT、build、schema、容量、同一origin検査を行う。全ての外部`uses:`は検証済み40桁commit SHAへ固定し、tag、branch、短縮SHAをfail-closedで拒否する。deploy jobはmainの成功時だけ実行し、`pages: write`と`id-token: write`だけを追加する。取得・VOICEVOX生成はCI通常経路から呼ばない。Nodeは`.nvmrc`とActions設定で同じ版へ固定し、`npm ci`だけを使用する。承認前検査は本番repositoryがprivateかつPages未有効であるfreshな読取証跡と、不変のrelease commit/catalog hashを持つ`ReleaseVisibilityPlan`だけを検査し、本番のpublic化やPages有効化を行わない。結果は`ready_for_approval | blocked`とする。ゲート④承認後のrelease処理だけが、承認直後かつpublic化直前にprivateを再観測し、public化後にPagesを有効化する。事後検査はclosedかつ回答`承認`のapproval ID、承認対象commit、GitHub visibility audit event、artifact digest、deployment ID、Pages hashを同一release chainとして`ReleaseVisibilityEvidence`で結合し、`released | blocked`を返す。承認前public化、Pages先行、hash chain不一致、順序や時刻の欠落は公開失敗とする。リリース証跡にはhosted Actionsのrun URL、artifact digest、deployment IDと、非main/失敗runでdeployが作成されず既存Pages hashが不変である確認を保存する。

### DES-F001-017: 再現可能なコンテンツ更新 [REQ-F001-029]

更新は`書誌取得 → 作品選定 → 本文raw bytes取得 → provenance → charset decode → extract → normalize → review gate → voice preflight → voice → public build`の11段階とし、中間物、入力hash、ツール版を残す。青空取得は公式HTTPS originと用途別path prefixを固定し、redirect・環境proxyを無効化し、接続前に名前解決結果のprivate/link-local/loopbackを拒否する。本番transportは検証済みaddressへ接続しつつ、TLS証明書とhostnameを公式hostに対して検証する。書誌1件と各作品を対象ごとに直列で1要求だけ取得し、自動retryしない。試験時も本番transportを使い、実ネットワークへ出ないfake resolverとpinned socket/TLS factoryだけを注入して、同一のURL・Host/SNI・証明書検証・接続先・redirect/proxy無効化契約を観測する。通常buildはネットワークを無効にし、RFC 3339時刻を固定して同じ公開hashを再現する。各段階は実体pathを検証した一時directoryへ出力する。公開treeは全参照検査済みstaging directoryをdirectory swapし、swap途中失敗時は旧treeを復元してcatalog/provenance/audioの組合せを混在させない。

### DES-F001-018: ブラウザ・権利リリース判定 [REQ-F001-030]

installed stableのChrome、Edge、Firefox、Safari、Android Chrome、iOS Safariについてトップ→作者→再生→一時停止→再開→停止→先頭から再生、レスポンシブ、CSP、同一origin、クレジットを確認する。CIのChromium/Firefox/WebKitは事前回帰に使うが、installed browserやmobile実機の代替とは扱わない。実施者はプロジェクト所有者または明示委任された検証者とし、`docs/evidence/qt/QT-F001-browser-manual.md`へ端末/OS/browser版、実施日時、各手順、画面証拠、判定者を6行すべて記録する。いずれかを用意できない場合はQキューへ検証協力または要求変更をエスカレーションし、証跡なしでPASSにしない。ライセンス規約確認もRFC 3339の確認日時・URL・判定者を残す。

### DES-F001-019: エラー処理と診断境界 [REQ-F001-008][REQ-F001-012][REQ-F001-015][REQ-F001-025]

更新工程の入力不正、括弧不整合、hash不一致、参照欠損とVOICEVOXのstage致命異常は理由コード付きで非0終了し、直前の正常な公開物を変更しない。台詞単位のVOICEVOX失敗は全体終了にせず理由付き公開除外として記録し、成功台詞とレビュー済み文字artifactを維持する。閲覧時のcatalog取得失敗は再読込導線を持つページ内エラー、音声失敗はカード内エラーに分離する。診断へ台詞本文以外の秘密情報や任意環境変数を保存しない。
