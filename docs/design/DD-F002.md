# 関数設計書 F002（継続コンテンツ拡充）

## 0. 型・artifact・path契約

### 0.1 主要型

| 型 | 必須契約 |
|---|---|
| `BatchId` | `^F[0-9]{3}$`を満たすbrand文字列。filesystem path断片へ使う前にも再検証する |
| `BatchStatus` | `draft | rights-verified | sources-fixed | extracted | reviewed | budget-approved | voiced | accepted | published`。`reviewed`以降は全`WorkProgress`の最低状態から導出する |
| `WorkStatus` | `pending | extracted | reviewed | budget-approved | voiced | accepted`。templateは全workを`pending`で生成し、manifest順で直前workがacceptedの場合だけ次workをreviewed以降へ進める |

| `BatchManifest` | `batchId, feature, schemaVersion, status, author, workIds[3], workProgress[3], inputPaths, outputPaths, stageRecords, rightsSnapshotIds, voiceConfigRef, artworkProvenanceRef, acceptedAt?, acceptedBy?, publishedAt?, releaseVersion?, deploymentEvidenceRef?, smokeEvidenceRef?`。accepted以降はaccepted項目、publishedは全publish項目を必須とする |
| `StageRecord` | `stage, inputHashes, toolVersion, outputHashes, count, completedAt`。canonical JSONをSHA-256化する |
| `StageEvidence` / `WorkTransitionEvidence` | 通常遷移は`kind='stage'`のStageEvidence、accepted遷移は`kind='accepted'`のPreparedWorkAcceptanceEvidenceを使うdiscriminated union。型を弱めて相互変換しない |

### 0.3 生成側・消費側path契約

KB-0001の教訓に従い、関数間で渡すpathは次の表以外の基準を使わない。

| artifact | 生成側 | 保存表現 | 消費側 | 消費時の基準 |
|---|---|---|---|---|
| batch定義 | `writeBatchManifestAtomic` | `content/batches/{batchId}/batch.json`（workspace相対POSIX） | `loadAcceptedBatches` | workspace絶対pathへ1回だけresolve |
| 原典・中間物 | batch stage runner | `data/batches/{batchId}/...`（workspace相対POSIX） | 次stage runner | manifest記載pathをworkspaceにresolve |
| 規約raw cache | `fetchPolicyObservation` | `.cache/rights/{batchId}/{policyId}/{sha}.snapshot` | 明示rights reviewだけ | workspace基準。gitignoreしpublic/Actions artifactへ含めない |
| review | review stage | `content/batches/{batchId}/reviews/{workId}.json` | `applyWorkReviews` | workspace基準 |
| voice cache | `generateVoiceDiff` | `.cache/voice/{configHash}/{audioId}.wav` | voice stage | workspace基準。public pathへ直接使わない |
| accepted音声source | `promoteVerifiedWorkArtifacts` | `content/batches/{batchId}/accepted-audio/{workId}/{audioId}.wav`（workspace相対POSIX、非ignore） | `buildIntegratedPublicTree` | 受入時はcommit候補、prepare/release時はsourceCommit/HEADでGit追跡済みかつmanifestのpath/SHA/bytes一致を検証。cacheを代用しない |
| 公開音声 | `buildIntegratedPublicTree` | `audio/{batchId}/{audioId}.wav`（公開root相対POSIX） | `resolvePublicAsset` | `new URL(path, publicBaseUrl)` |
| 作者画像 | artwork stage | `artwork/{slug-or-fixed-name}.png`（公開root相対POSIX） | browser/credits | `resolvePublicAsset` |
| 公開catalog | public build | `content/catalog.json`（公開root相対POSIX） | `loadCatalog` | `publicBaseUrl` |
| staging/public | public build | filesystem絶対path | `promoteIntegratedTree` | workspace実体path・reparse拒否 |

公開root相対pathは先頭slash、空segment、`.`、`..`、backslash、scheme、authority、query、fragment、percent-encoded separator/controlを拒否する。青空文庫URLはexact origin、人物directory、work IDと許可path形式へbrand化し、credentials、任意port、query、fragment、protocol-relative、encoded dot segment、ID混線を拒否する。workspace相対pathはPOSIX表現でmanifestへ保存し、filesystem APIへ渡す直前にのみOS pathへ変換する。生成側と消費側を同じfixtureで通す結合試験を必須とする。

## 1. batch・catalog・baseline

### FUN-F002-001: validateBatchManifest(value: unknown) -> ValidationResult<BatchManifest> [DES-F002-002][DES-F002-014]

- **追加先**: 新規`src/content/batch.ts`。`scripts/content-cli.ts`の固定path群をこの結果で置換する。
- **入力**: JSON parse済みの未知値。
- **出力**: brand済み`BatchId`、一意なauthor/work IDs、ちょうど3 work、work ID順と一致する`workProgress`、許可された相対path、正しい状態・stage hash chainを持つmanifest。同一authorの後続batchは期待identity hashを必須とする。`accepted`は`acceptedAt/acceptedBy`、`published`はそれらに加えて`publishedAt/releaseVersion/deploymentEvidenceRef/smokeEvidenceRef`を必須とし、それ以前の状態にpublish項目があれば拒否する。
- **エラーcode**: `BATCH_SCHEMA_INVALID`、`BATCH_ID_INVALID`、`BATCH_WORK_DUPLICATE`、`BATCH_PATH_INVALID`、`BATCH_STAGE_HASH_MISMATCH`。F002ではauthor `000081`、work IDs `000473/043752/043754`の順序もfixture manifestと一致させる。

### FUN-F002-002: transitionBatchState(manifest: BatchManifest, next: BatchStatus, evidence: StageEvidence) -> BatchManifest [DES-F002-002][DES-F002-015]

- **追加先**: `src/content/batch.ts`。
- **入力**: 現manifest、次状態、入力/出力hash・件数・tool版・時刻を持つbatch証跡。work状態変更にはFUN-F002-031だけを使う。
- **出力**: `rights-verified/sources-fixed/extracted`の前段batch状態、または全workの最低状態から導出した`reviewed/budget-approved/voiced/accepted`へ更新し、新しい`StageRecord`を末尾追加したimmutable値。accepted遷移では`acceptedAt/acceptedBy`を証跡から設定し、`published`とpublish項目はFUN-F002-037だけが設定する。
- **状態条件**: 全workが各状態へ達したときだけ同名batch状態を導出する。`pending`を含む間はrights/source stageに応じたbatch前段状態へ写像し、先行workだけがacceptedでも後続work処理を妨げない。
- **エラーcode**: `BATCH_STATE_SKIP`、`BATCH_STATE_REWIND`、`BATCH_GATE_INCOMPLETE`、`BATCH_EVIDENCE_STALE`。元manifestは変更しない。

### FUN-F002-003: writeBatchManifestAtomic(workspace: AbsolutePath, manifestPath: WorkspaceRelativePath, next: BatchManifest, expectedSha256: Sha256) -> Promise<Sha256> [DES-F002-002][DES-F002-015]

- **追加先**: `src/content/batch.ts`。既存`writeJsonArtifactAtomic`を利用する。
- **入力**: 実体確認済みworkspace、`content/batches/{batchId}/batch.json`、更新前canonical hash、検証済みmanifest。
- **出力**: UTF-8 canonical JSONを同directory一時fileへ書き、`FileHandle.sync()`、競合再確認後rename、再open/hash検証した新hash。directory sync対応OSではrename前後に実行する。
- **エラーcode**: `BATCH_WRITE_CONFLICT`、`BATCH_WORKSPACE_BOUNDARY`、`BATCH_FILE_SYNC_FAILED`、`BATCH_ATOMIC_RENAME_FAILED`、`BATCH_POSTWRITE_MISMATCH`。symlink/junction/reparse、更新前hash不一致では旧fileを維持する。Windowsでdirectory sync非対応の場合は能力をstage evidenceへ記録し、file sync・rename・postwrite hash・recovery journalを必須とする。既存helperはこの契約へ拡張してから再利用する。

### FUN-F002-004: validateCatalogV2(value: unknown, sourceBytes: number) -> ValidationResult<UICatalogV2> [DES-F002-001][DES-F002-006][DES-F002-012]

- **置換先**: `src/ui/catalog-loader.ts::validateCatalog`と`src/content/processing.ts`の公開catalog検査。
- **入力**: raw UTF-8 byte数を測定後にparseしたJSON。
- **出力**: 1作者以上を含み、ID/slug一意、author→work→dialogue→audio→batchの全参照、source/asset path、候補集計、全authorに1作品以上を検証した`CatalogV2`。F002受入の2作者以上条件はFUN-F002-030で検証する。
- **エラーcode**: `CATALOG_RESOURCE_LIMIT`、`CATALOG_DUPLICATE_ID`、`CATALOG_ORPHAN_REFERENCE`、`CATALOG_AUTHOR_MIXED`、`CATALOG_PATH_UNSAFE`、`CATALOG_COUNT_MISMATCH`。特定作者ID、作品数3、Aozora人物pathの固定条件を持たない。

### FUN-F002-005: loadAndVerifyF001Baseline(sourceRoot: AbsolutePath, baseline: F001Baseline) -> Promise<F001BaselineBundle> [DES-F002-003][DES-F002-006][DES-F002-016]

- **追加先**: 新規`src/content/baseline.ts`、`scripts/verify-project.mjs`から呼ぶ。
- **入力**: clean cloneでも存在する追跡済みF001 source root、`content/baselines/F001-v0.1.0.json`。CatalogV2を入力せず、source rootを暗黙cwdから推定しない。

## 2. 原典・レビュー・権利

### FUN-F002-006: selectBatchWorks(rows: BibliographyRow[], manifest: BatchManifest, observedAt: Date) -> SelectedWorkResult [DES-F002-004][DES-F002-009]

- **置換先**: `src/content/source.ts::selectEligibleWorks/resolveEdition`の固定`INITIAL_*`呼出を汎用化し、`src/content/production.ts`から利用する。
- **入力**: schema検証済み公式拡充CSVとmanifestのauthor/work allowlist。
- **出力**: manifest順の3作品と`phase=selection`の`WorkRightsObservation`。人物ID、著者、人物/作品著作権なし、翻訳者なし、公開中、新字新仮名、図書カード/本文URL、公式書誌hash・取得時刻を保持する。
- **エラーcode**: `WORK_RIGHTS_INELIGIBLE`、`WORK_ROLE_INVALID`、`WORK_TRANSLATOR_PRESENT`、`WORK_EDITION_AMBIGUOUS`、`WORK_ALLOWLIST_MISMATCH`。

### FUN-F002-007: runBatchSourceStages(context: BatchContext, through: 'normalize') -> Promise<BatchSourceResult> [DES-F002-004][DES-F002-014][DES-F002-015]

- **置換先**: `src/content/production.ts`の`INITIAL_WORK_IDS`、固定title/path、F001 evidence pathをBatchContextへ置換する。
- **入力**: workspace、BatchManifest、安全なAozora transport、固定clock、speech rules。
- **出力**: `data/batches/{batchId}/bibliography|selected-works|sources|intermediate`と`content/batches/{batchId}/provenance.json`。同じraw bytes・tool版から候補順、candidate ID、hashが一致する。
- **状態更新**: 各workの抽出・正規化artifactがhash検証を完了した時だけFUN-F002-031へ証跡を渡し、そのworkを`pending`から`extracted`へ進める。未処理workは`pending`のままにする。
- **エラーcode**: 既存のTLS/ZIP/charset/DOM/bracket/hash codeに`BATCH_*_MISMATCH`を加える。各stageを一時treeへ書き、成功時だけ対象batch treeをatomic promotionする。

### FUN-F002-008: normalizeBatchCandidate(raw: RawCandidate, rules: SpeechRules, prior?: SpeechRevision[]) -> CandidateWithRevisions [DES-F002-004]

- **追加/置換先**: `src/content/processing.ts`の`normalizeDisplayText/normalizeSpeechText/createCandidateId`を合成するwrapper。
- **入力**: ruby tokenを含む候補、版付き規則、任意の既存補正履歴。
- **出力**: 原典意味を保つdisplay text、決定的speech text、既存と同じID tuple、最新speech textと一致するrevision履歴。
- **エラーcode**: `DISPLAY_TEXT_MUTATED`、`SPEECH_REVISION_GAP`、`SPEECH_REVISION_MISMATCH`、既存`NormalizationError`。旧候補IDへのreviewは自動転用しない。

### FUN-F002-009: applyWorkReviews(workId: WorkId, candidates: Candidate[], reviews: ReviewRecord[]) -> WorkReviewResult [DES-F002-004][DES-F002-015]

- **置換先**: `src/content/production-final.ts`の固定3 review artifact loaderと`applyEditorialReview`呼出。
- **入力**: manifest内の単一work ID、そのworkの全候補と対応review履歴。後続work候補を入力しない。
- **出力**: 当該workのapproved/rejected/pending=0、理由集計、revision、reviewer、日時、policy判断。全候補は厳密に1分類される。batch集計は各`WorkProgress`から導出し、後続workのpendingを許容して先行workの遷移を妨げない。
- **エラーcode**: `REVIEW_WORK_MISMATCH`、`REVIEW_MISSING`、`REVIEW_DUPLICATE`、`REVIEW_ORPHAN`、`REVIEW_REVISION_CONFLICT`、`REVIEW_REASON_INVALID`、`REVIEW_PENDING`。`REVIEW_PENDING`は当該work内だけを対象とし、文学表現だけを理由に自動拒否しない。

### FUN-F002-010: capturePolicyObservation(policy: PolicyDefinition, response: FetchedPolicyResponse, now: Date, reviewer: string, decisionSummary: string) -> PolicyObservation [DES-F002-009][DES-F002-012]

- **追加先**: 新規`src/notices/policy-snapshots.ts`。明示`rights-snapshot` stageだけがFUN-F002-035の結果を渡す。
- **入力**: 用途別allowlist済み定義、安全transportと結合済みresponse、固定可能なclock、reviewer、人が読める判定要約。
- **出力**: final URL・status・media type・response bytes・transport版・raw SHA-256・版ラベル・判定要約を持つ観測値。raw bytesはgitignore済み`.cache/rights/{batchId}/{policyId}/{sha}.snapshot`へatomic保存し、commit/public/Pages/Actions artifactには含めない。
- **エラーcode**: `POLICY_RESPONSE_UNBOUND`、`POLICY_MEDIA_INVALID`、`POLICY_TOO_LARGE`、`POLICY_SNAPSHOT_WRITE_FAILED`。通常buildとbrowserからは呼ばず、公開tree検査はsnapshot混入を拒否する。

### FUN-F002-011: comparePolicySnapshots(selection: PolicyObservation[], predeploy: PolicyObservation[], reviews: ImpactReview[]) -> PolicyDecision [DES-F002-009][DES-F002-015][DES-F002-016]

- **追加先**: `src/notices/policy-snapshots.ts`、release checksへ接続。
- **入力**: policy IDごとの選定時・deploy直前観測と、hash差分時の影響再レビュー。

- **エラーcode**: `POLICY_OBSERVATION_MISSING`、`POLICY_HASH_CHANGED_UNREVIEWED`、`POLICY_REVIEW_STALE`、`POLICY_ID_MISMATCH`。未判定は常にblocked。

### FUN-F002-012: validateArtworkProvenance(manifest: ArtworkProvenanceV2, workspace: AbsolutePath) -> Promise<ArtworkDecision> [DES-F002-010][DES-F002-012]
