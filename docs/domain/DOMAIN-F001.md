# ドメイン調査書 F001（青空文庫朗読・静的音声サイト）

## Tier判定

**大**。一般公開サイトで第三者の本文・書誌データ・キャラクター・音声合成物を扱い、著作権、キャラクター利用条件、クレジット、GitHub Pagesの容量・商用制約を同時に満たす必要があるため。

## 結論

- 初期公開は、広告・課金・スポンサー表示のない個人運営の非公式ファンサイトとする。
- 青空文庫の全項目を一律に使わず、公式書誌データで`人物ID=000879`、`役割=著者`、`作品著作権=なし`を満たす芥川龍之介の日本語原著だけを候補にする。
- 同一作品の版違いは原則として新字新仮名を優先し、現代英訳`Afterlife`と芥川が翻訳者である作品を除外する。
- 実行時に青空文庫やVOICEVOXへ接続しない。更新処理で本文取得、台詞候補の確認、VOICEVOX音声生成を行い、JSONと圧縮済み音声をGitHub Pagesへ静的配信する。
- 音声クレジット`VOICEVOX:ずんだもん`を全ページのフッターとクレジットページに表示する。
- 初期サムネイルは、坂本アヒル氏の配布素材V3.2を利用条件の範囲で改変する案を第一候補とする。公開物に元PSDを含めず、`立ち絵：坂本アヒル`を表示する。素材取得時のREADMEとハッシュを保管する。
- ずんだもんのキャラクター利用と青空文庫本文の利用は別の権利判断である。初期版を無料・広告なしに限定し、収益化前にSSSの最新商用条件を再確認する。

## 対象データ

2026年7月17日時点の青空文庫拡充版UTF-8 CSVでは、人物ID`000879`の379件中、著者として登録されたものが374件、そのうち作品著作権フラグ「なし」が373件である。ただし版違いの重複があり、すべてが読み上げ対象になるわけではない。

| 除外条件 | 理由 |
|---|---|
| 作品著作権フラグが「あり」 | 原著が保護期間満了でも翻訳等の権利が存続し得る |
| 役割が「翻訳者」 | 「芥川龍之介の作品」という初期サイトの対象外 |
| 公開中でない項目 | 現行の公式公開状態を優先する |
| 明示的に同一作品と確認した旧字版等 | 音声と一覧の重複を避け、新字新仮名を優先する |
| 台詞候補がない作品 | 音声ボタンを生成できない |

青空文庫の保護期間満了ファイルは、有償・無償を問わず複製、再配布、口述、翻案が可能と案内されている。芥川龍之介は1927年没で、日本では2018年の保護期間延長前に権利が消滅しており、消滅した権利は復活しない。ただし日本国外での提供は各国法の評価が別途必要になり得る。

## 出典・加工履歴

各作品には次を表示または追跡可能にする。

- 作品名、著者名、安定した図書カードURL
- 底本、初出、入力者、校正者、ファイル作成日・更新日
- 青空文庫由来である旨
- 「会話括弧内を抽出し、ルビ・注記を読み上げ用に正規化した」旨
- 取得元URL、取得日、原文ハッシュ、変換器バージョン

同名タイトルだけでは重複判定しない。たとえば「仙人」3件は同名の別作品であり、すべて保持する。版統合は本文と図書カードを確認した作品IDの明示的な対応表でのみ行い、新字新仮名がない作品は他の表記種別へfallbackする。

書誌CSVを利用する箇所は`書誌データ：青空文庫（CC BY 4.0）`と表示し、ライセンスへのリンクと、このサイトで選定・整形した変更内容を明記する。これはCC BY 4.0の条件として扱う。本文そのものをCC BY 4.0と誤表示しない。本文の由来情報保持は青空文庫の希望・期待として尊重し、表示原文と読み上げ用正規化文を分離して原文の意味を改変しない。

## 台詞抽出

単純な正規表現だけでは`「」`内の発話、内心、引用、語句例を区別できない。XHTMLの本文DOMを入力とし、次の二段階処理を採用する。

1. 改行と`『』`の入れ子を考慮した走査で`「」`内を候補抽出する。
2. 周辺の「言った」「答えた」等を加点し、「思った」「という語」等を除外候補としたうえで、固定allowlistによる公開前確認を行う。

表示用本文と読み上げ用本文を分離する。読み上げ用では`ruby/rt`を読みへ変換し、外字・注記・傍点・空白を決定的に正規化する。自動判定の確信度が低い候補は公開しない。

## 音声・キャラクター利用条件

- 生成済みVOICEVOX音声のWeb配信は、VOICEVOX本体とずんだもん音源の両規約に従う。
- `VOICEVOX:ずんだもん`を利用者が容易に確認できる位置へ表示する。
- 文学作品の発話であること、作品名、著者名を併記し、ずんだもん自身の主張と誤認させない。
- 政治・宗教活動、特定人物・団体への批判・応援、情報商材、公序良俗違反等の禁止用途に該当する候補を自動公開しない。
- 坂本アヒル氏のV3.2素材を使う場合は配布README、SSSキャラクターガイドラインの双方に従う。商用利用を無条件とは扱わず、地域・主体・用途によるSSS側の条件を優先する。改変画像だけを配信し、素材データの再配布ページにしない。
- 芥川龍之介の特定写真は入力・合成せず、一般的な大正期文豪の髪型、外套、本等で表現する。写真を使う場合は写真家・所蔵者を含む個別の権利証跡を別途要求する。
- サイトへ「東北ずん子・ずんだもんプロジェクトの非公式ファンサイトです」と表示する。
- 規約は変更され得るため、リリースゲートで再確認する。

## 静的配信アーキテクチャ

推奨フローは次のとおり。

1. 手動または明示的なコンテンツ更新ジョブで公式CSV/XHTMLを取得する。
2. 対象条件、重複排除、台詞抽出、レビュー用allowlistを適用する。
3. 正規化本文、音声スタイル、VOICEVOX ENGINE版を含むハッシュで音声IDを決める。
4. ローカルまたは固定バージョンのVOICEVOX ENGINEで未生成分だけ合成する。
5. 音声をモノラルの圧縮形式へ変換し、JSON、出典、画像とともに静的ビルドする。
6. 通常のPagesデプロイでは外部TTS・青空文庫へ接続せず、同一オリジンの音声をクリック時だけ取得する。

ブラウザから利用者のローカルVOICEVOX ENGINEへ接続する方式は、CORS設定、ローカル導入、再起動、端末差を利用者へ要求するため採用しない。2026年7月18日時点で、利用者のローカルENGINEを不要とする公式サポート済みブラウザ単体構成を確認できないため、大容量モデルをブラウザへ同梱する方式も初期版では採用しない。

## GitHub Pages・容量

- 公開時はGitHub FreeでPagesを使えるようリポジトリをpublicへ変更する。
- project siteのbaseを`/bungo-zundamon/`へ固定し、hash routeまたは実在ファイルだけを使う。
- 公開成果物は500MB未満を目標、750MBを公開停止基準とする。GitHub Pagesの公開サイト上限1GBへ接近させない。
- 音声は作品単位・台詞単位に分割し、`preload="none"`で選択時だけ取得する。Git LFSはPages配信に使わない。
- Actionsのbuildは`contents: read`、deployは`pages: write`と`id-token: write`だけを付与し、検査済みmainからのみ公開する。
- 初期版は解析、広告、認証、課金、フォーム、個人情報収集を持たない。

## セキュリティ・アクセシビリティ

- 取得したXHTMLをそのまま`innerHTML`へ挿入せず、変換済みJSONをDOM APIまたはフレームワークの通常描画で表示する。
- CSPは同一オリジンのscript、style、image、audioだけを許可し、`object-src 'none'`、`base-uri 'none'`、`form-action 'none'`を設定する。
- 音声は利用者のボタン操作でのみ開始し、同時に一音声だけを再生する。停止、再生中表示、読込失敗からの再試行を提供する。
- すべての音声ボタンに作品名と台詞を含むアクセシブルな名前を付け、キーボード操作、フォーカス表示、44 CSS px以上の主要操作領域を用意する。
- `prefers-reduced-motion`を尊重し、演出は情報理解や操作を妨げない。

## 未確実点・QA候補

- 373件の候補すべてへ音声を生成するとPages容量、レビュー工数、リポジトリ規模が過大になる。初期公開の作品数と台詞数を決める必要がある。
- 将来、広告・有料サービス・企業販促を行う場合は、公開前にSSSの商用窓口とGitHub Pagesの商用制約を再確認する必要がある。
- 坂本アヒル氏素材の配布READMEを実装時に再取得し、取得時点の条件を証跡化する必要がある。
