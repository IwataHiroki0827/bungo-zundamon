# 文豪ずんだもん 要求メモ

## 1. 目的

青空文庫で公開されている著作権保護期間満了作品から、かぎ括弧`「」`で囲まれた口語の台詞を抽出し、ずんだもん音声で手軽に再生できる静的Webアプリ「文豪ずんだもん」を公開する。

## 2. 初期対象

- 対象作者: 芥川龍之介
- 青空文庫作者ページ: https://www.aozora.gr.jp/index_pages/person879.html#sakuhin_list_1
- サイト上の表示名: `あくたがわずんのすけ`
- 初期版は上記ページの「公開中の作品」のうち、著作権・入力データの利用条件を確認でき、台詞を抽出できる作品を対象とする
- 将来は他の著作権保護期間満了作者を追加できるデータ構造にする

## 3. 画面構成

1. 作者一覧
2. 作者詳細（初期版: あくたがわずんのすけ）
3. 作品・台詞一覧
4. 各台詞に音声再生ボタン、作品名、本文出典へのリンクを表示

スマートフォンとPCの両方で、一覧から迷わず音声再生できる操作にする。再生中・停止・読込失敗を視覚表示し、キーボード操作とスクリーンリーダーにも対応する。

## 4. 本文処理

- 青空文庫の配布テキストまたは公式HTMLをbuild時に取得・正規化し、実行時スクレイピングは行わない
- 青空文庫注記、ルビ、外字注記、底本情報を壊さず処理し、表示用台詞からは読み上げ不要の注記を除く
- 全角かぎ括弧`「...」`を対象とし、複数行、連続台詞、入れ子、空文字、極端に長い台詞の扱いを仕様化する
- 抽出結果は重複を管理し、作品名・作品URL・入力者/校正者等の必要クレジットへ追跡できるようにする
- 不正なHTMLを挿入せず、表示はテキストとして扱う

## 5. 音声

- ずんだもんが各台詞を読み上げる
- 初期案はVOICEVOX:ずんだもんで開発/build時に生成した静的音声を同梱し、GitHub Pages上で外部音声APIや秘密鍵を使わない
- ボタン操作時だけ再生し、自動再生しない
- 同時再生を防ぎ、連打・通信失敗・音声非対応ブラウザでも画面を停止させない
- VOICEVOX、ずんだもん、使用立ち絵の最新利用規約を一次情報で確認し、必要なクレジットを常設ページへ表示する

## 6. ビジュアル

- サムネイルと作者画像は、ずんだもんと芥川龍之介を融合した「あくたがわずんのすけ」のキャラクター
- ユーザ希望は坂本アヒル氏のずんだもん立ち絵を想起させる親しみやすい雰囲気
- 既存立ち絵の直接加工・再配布は配布元の商用利用・加工・クレジット条件を確認してから採用する
- 条件が不明または商用利用に適さない場合は、芥川龍之介の肖像要素と枝豆モチーフを用いた新規オリジナルキャラクターへ切り替え、特定イラストの複製を避ける
- 色だけに依存せず、十分なコントラストと代替テキストを用意する

## 7. 公開・運用

- GitHub Pagesで公開し、ランニングコストを0円にする
- 静的サイトとして構築し、サーバー、データベース、外部APIキーを不要にする
- 初期GitHubリポジトリはProjectFactory規約に従いprivateで作成し、公開直前にライセンス確認後publicへ変更する
- 相対パス、CSP、依存関係監査、認証情報検査、404確認を行う
- 青空文庫・VOICEVOX・キャラクター素材・画像・フォントの出典と利用条件を`CREDITS`/利用規約ページへ記載する

## 8. プロジェクト運用

- 種別: Webアプリ
- ProjectFactory ID: `bungo-zundamon`
- `run_mode: auto`
- 承認後自動再開: `approval_resume: auto`
- ユーザは作業中不在のため、発見可能な事項は一次情報と安全側の既定で解決する
- ProjectFactoryの要求・設計・テスト仕様・リリースの4承認ゲートは省略せず、成果物を確認できる形でキューへ登録する

## 9. 受け入れイメージ

- 作者一覧から「あくたがわずんのすけ」を選べる
- 作者詳細で作品ごとの台詞と音声ボタンが一覧表示される
- ボタン1回で対応するずんだもん音声が再生される
- スマートフォンで主要操作が画面内に収まり、キーボードでも利用できる
- 作品・音声・画像の出典とクレジットを確認できる
- GitHub Pagesの公開URLで、外部有料サービスなしに動作する

## 10. F002 継続コンテンツ拡充

### 10.1 目的

- 既存の芥川龍之介3作品と音声を維持したまま、作者・収録作品・VOICEVOX音声を継続的に追加できる構造へ拡張する
- 追加は「1作者・代表3作品」を1バッチとし、権利・品質・容量・セキュリティの確認を各バッチで完結させる
- 初回バッチの実績を、後続作者を同じ手順で追加できる自動化・文書・試験へ反映する

### 10.2 初回追加作者・作品

- 追加作者: 宮沢賢治（青空文庫人物ID `000081`）
- サイト上の表示名: `みやざわずんじ`（原著者名「宮沢賢治」を常に併記する）
- 対象作品は、青空文庫の新字新仮名版として公開中で、人物・作品の著作権フラグがともに`なし`の次の3作品とする
  1. `よだかの星`（作品ID `000473`）
  2. `どんぐりと山猫`（作品ID `043752`）
  3. `注文の多い料理店`（作品ID `043754`）
- 公式図書カード、本文URL、底本、入力者、校正者、取得日時、原典SHA-256を保持する

### 10.3 台詞・音声

- 既存と同じく全角かぎ括弧`「...」`を候補とし、候補を全件編集レビューして`pending: 0`にする
- 扉の注意書き、手紙、判決文、引用、入れ子、発話でない表示文は理由付きで除外する
- 文学作品中のいじめ・死・暴力・宗教・侮辱表現だけを理由に実発話を機械除外しないが、作品中の表現であることと出典を明示する
- 表示文とVOICEVOX読み上げ文を分離し、固有語・擬音・独特な表記の読み補正履歴を残す
- 承認済み台詞だけを`VOICEVOX:ずんだもん`で差分生成し、失敗0件、WAVのSHA-256・bytes・durationを記録する
- 実行時は同梱した静的音声だけを再生し、VOICEVOXその他の外部音声APIへ通信しない

### 10.4 複数作者UI

- 作者一覧に芥川龍之介と宮沢賢治を表示し、各作者を選択して固有URLへ直アクセスできる
- route、catalog、出典link、画像、作品数・台詞数を作者データから解決し、特定作者・3作品へのハードコードを解消する
- 作者ごとの作品・台詞・音声を混線させず、再生中に作者または画面を切り替えたら既存音声を停止する
- 新作者画像は独自に作成・生成したもの、または利用条件を確認した素材だけを使い、第三者の二次創作を無断利用しない

### 10.5 容量・継続運用

- 音声生成前に作業ドライブの空き容量、追加音声見積り、公開総容量を検査する
- 1バッチの追加音声は100 MiB以下、公開総容量500 MiB以上で警告、750 MiB超の見込みでは生成・公開を停止して配信方式を見直す
- cacheは音声ID単位で再利用し、同一入力・同一VOICEVOX設定の音声を重複生成しない
- 後続バッチもProjectFactoryで新フィーチャーとして起票し、同じ権利確認、全候補レビュー、容量preflight、試験、承認ゲートを通す
- 次回候補は宮沢賢治「雪渡り」、次作者候補は太宰治とし、現行の`「」`抽出で台詞が得られにくい「やまなし」は括弧仕様拡張まで見送る

### 10.6 品質・セキュリティ

- 既存337件の自動試験と78件のブラウザ試験を回帰基準とし、複数作者・新作品・新音声の試験を追加する
- XSS対策、同一origin asset、CSP、外部通信0、Cookie/storage/form不使用、依存関係監査0件、秘密情報検査を維持する
- 青空文庫、底本・入力者・校正者、VOICEVOX、ずんだもん、作者画像の出典・利用条件をクレジット画面から確認できる
- スマートフォン、キーボード、スクリーンリーダー、reduced motion、音声エラー隔離、同時再生防止でF001の品質水準を下げない
